## 二、系统方案设计

### 系统总体架构

本系统采用三层异构分层架构，实现 AI 视觉识别与底层设备控制解耦，通过 USB CDC 虚拟串口完成上下位机数据交互：

1. **感知层（上位机：昇腾 310B）**：完成摄像头图像采集、YOLOv10 手势 NPU 推理、视频编码推流，输出手势识别数据。

2. **传输层（USB CDC 虚拟串口）**：替代传统物理串口，单 USB 线同时实现供电与数据双向传输，连接灵活、抗干扰性强。

3. **执行层（下位机：STM32）**：接收解析手势指令，输出 PWM 波形，驱动风扇、舵机执行对应动作。

整体流转逻辑：
图像采集 → AI 手势推理 → 虚拟串口传输指令 → STM32 指令解析 → 外设执行控制

### 硬件框图

```
USB摄像头 ──→ 昇腾310B (NPU推理) ──USB CDC──→ STM32F103 ──PWM──→ L298N → 风扇
                                         │                    ├─ SG90#1 → 左右摆风
                                         │                    └─ SG90#2 → 上下摆风
                                    WebRTC → 浏览器
```

### 软件流程

**310B**：取帧 → Letterbox 预处理 → NPU 推理 → 后处理 → 分支1：USB CDC 发 JSON / 分支2：VENC 推流

**STM32**：USB CDC 中断接收 → JSON 解析 → 查表匹配手势 → TIM3/TIM4 PWM 输出 → 风扇转速/舵机角度

## 四、详细设计文档

### 硬件部分

#### 1\. 电路连接逻辑

- 上下位机通信：昇腾 310B（USB Host）→ USB 数据线 → STM32 PA11/PA12（USB Device，CDC 虚拟串口）

- 外设驱动接线：
STM32 PA6 → L298N ENA（风扇调速 PWM）
STM32 PB3/PB4 → L298N IN1/IN2（风扇转向）

- L298N 供电规范：VSS 接 5V 逻辑电，VS 外接 12V 动力电源

#### 2\. PCB 布局设计规范

1. 功率器件布局：L298N 驱动、电机接线端子放置板边，大面积铺铜提升散热；

2. 电磁兼容优化：USB 通信走线严格阻抗匹配，远离电机大电流回路，降低 EMI 干扰，避免数据包丢失、乱码。

### 软件部分

#### 1\. 软件模块分层架构

**昇腾 310B 上位机模块**
摄像头采集模块 → YOLO 模型推理模块 → 检测结果后处理模块 → 虚拟串口通信管理模块 → WebRTC 视频推流模块

**STM32 下位机模块**
USB 虚拟串口接收模块 → JSON 指令解析模块 → PWM 电机 / 舵机驱动控制模块

#### 2\. 核心算法流程

原图输入 → Letterbox 等比例缩放 \+ 灰色填充 → YOLOv10 无 NMS 推理输出检测框 → 坐标逆缩放 / 偏移校正 → 输出原图匹配的手势检测结果

#### 3\. 外设与通信配置

- 昇腾 310B 端：DVPP 硬件单元完成图像缩放、色彩转换；CDC\_VCOM 虚拟串口，参数 8N1（8 数据位、1 停止位、无校验）

- STM32 端：移植 USB CDC 协议栈，配合 DMA \+ 中断接收，保障高速数据稳定传输

#### 4\. 系统任务调度方案

- 昇腾 310B（Linux 多线程）：采集 / 推理 / 推流线程绑定独立 CPU 核心，串口发送线程设实时优先级，无锁环形队列传递图像数据，减少内存拷贝；

- STM32（FreeRTOS 规划）：电机控制任务最高优先级；USB 接收依靠硬件中断不占用 CPU；指令解析为普通优先级，消息队列同步控制信号。

### （二）待完善后续开发工作

1. 业务控制逻辑接入：在 STM32 主循环`g_gesture_ready`分支，根据手势 ID 编写风扇调速、舵机摆风控制代码；

2. 传输稳定性优化：增加 CRC 数据校验，兼容手部遮挡、光线差等异常场景的 JSON 解析容错；

3. 调试功能拓展：启用预留 USART1 串口，打印调试日志，方便故障定位；

4. 传输性能调优：按需降低手势上报频率、删减冗余检测框字段，降低带宽占用。
