## 一、Pi 端：消抖发送（camera.py，新增）

### GestureDebouncer 类

```python
class GestureDebouncer:
    def __init__(self, threshold=3):
        self.threshold = threshold  # 确认所需连击帧数
        self.candidate = None       # 当前候选手势 ID
        self.count = 0              # 连击计数
        self.last_sent = None       # 已发送的 ID（防重复）

    def update(self, class_id):
        # ── 1. 手势变了 → 候选重置 ──
        if class_id != self.candidate:
            self.candidate = class_id
            self.count = 1
            return False

        # ── 2. 相同手势 → 连击 +1 ──
        self.count += 1

        # ── 3. 满阈值 且 没发过 → 确认出口 ──
        if self.count >= self.threshold and class_id != self.last_sent:
            self.last_sent = class_id
            return True

        return False  # 不确认
```

**设计思路**：
- 摄像头 10Hz，同一手势持续时有 30+ 帧相同 class_id
- 前 3 帧做"连击确认"（滤掉单帧误识别），第 3 帧后标记 `last_sent`，后续相同 ID 直接跳过
- 手势切换时 `candidate` 变化，计数器重置，开始新一轮确认
- threshold=3 ≈ 300ms 延迟，跟手性与滤噪平衡

## 二、STM32 端：USB CDC 接收（usbd_cdc_if.c）

### JSON 行接收

```c
static int8_t CDC_Receive_FS(uint8_t* Buf, uint32_t *Len)
{
    for (uint32_t i = 0; i < *Len; i++) {
        if (Buf[i] == '\n') {               // 遇换行 → 完整一行
            rx_line[rx_idx] = '\0';
            parse_line(rx_line);            // 解析 JSON
            rx_idx = 0;
        } else if (rx_idx < 127) {
            rx_line[rx_idx++] = Buf[i];     // 逐字节累积
        }
    }
    USBD_CDC_SetRxBuffer(&hUsbDeviceFS, &Buf[0]);
    USBD_CDC_ReceivePacket(&hUsbDeviceFS);  // 重新注册（关键！漏了收不到下一包）
    return (USBD_OK);
}
```

### 极简 JSON 解析（不依赖库）

```c
// 从 {"c":21,"n":"peace","s":0.95} 中提取字段
r.class_id = json_get_int(line, "c");        // → 21
json_get_str(line, "n", r.class_name, 24);   // → "peace"
r.score = json_get_float(line, "s");          // → 0.95
```

`json_get_int` 原理：在字符串中找 `"c":` → 跳过 → `atoi` 转换。STM32F103C8T6 只有 20KB RAM，不能引入完整 JSON 库，所以手写了三个提取函数。

### 左右舵机：预判式折返（新方案）

```c
// 在主循环中每 10ms 执行一次
if (servo_lr_on && (HAL_GetTick() - servo_tick_lr >= 10)) {
    servo_tick_lr = HAL_GetTick();

    // ★ 先算下一步，不直接改 CCR
    int32_t next = (int32_t)servo_lr_ccr + servo_lr_dir * 2;

    if (next > 1150) {
        // 预判会越上界 → 不等，强制弹离
        servo_lr_ccr = 1150 - (next - 1150);
        servo_lr_dir = -1;
    } else if (next < 350) {
        // 预判会越下界 → 强制弹离
        servo_lr_ccr = 350 + (350 - next);
        servo_lr_dir = 1;
    } else {
        // 正常步进
        servo_lr_ccr = (uint16_t)next;
    }
    __HAL_TIM_SET_COMPARE(&htim4, TIM_CHANNEL_1, servo_lr_ccr);
}
```

**与旧方案的区别**：

| | 旧方案（先加后判） | 新方案（预判式） |
|------|------|------|
| 顺序 | ccr 先加上，到了再弹 | 先算 next，越界当场弹 |
| 边界行为 | 会在边界值上停一个 tick | 永远不会停在边界 |
| 效果 | 端点肉眼可见微小停顿 | 理想情况下丝滑 |

### 双舵机独立 tick

```c
static uint32_t servo_tick_lr = 0;   // 左右独立计时
static uint32_t servo_tick_ud = 0;   // 上下独立计时
```

旧方案共享一个 `servo_tick`，两轴同时扫时互锁导致节奏混乱。拆开后互不干扰。

## 四 usb_sender.py

```python
class USBSender:
    def __init__(self, port='/dev/stm32', baudrate=115200):
        self.count = 0
        self.last_time = 0
        self.ser = None
        try:
            import serial
            self.ser = serial.Serial(port, baudrate, timeout=0.1)
            print(f"[USB CDC] {port} 已连接")
        except Exception as e:
            print(f"[USB CDC] 失败: {e}")

    def send_detection(self, detection):
        """发送识别结果到 STM32 — JSON 行协议"""
        if detection.get('score', 0) < 0.5:
            return
        cid = detection.get('class_id', -1)
        name = NAMES.get(cid, f"ID{cid}")
        payload = {"c": cid, "n": name, "s": round(detection.get('score', 0), 4)}
        bbox = detection.get('bbox', None)
        if bbox and len(bbox) == 4:
            payload.update(x=int(bbox[0]), y=int(bbox[1]), w=int(bbox[2]), h=int(bbox[3]))

        line = _json.dumps(payload, ensure_ascii=False) + '\n'
        now = time.time()
        if now - self.last_time < 0.1:
            return
        self.last_time = now
        self.count += 1
        print(f">>> [USB #{self.count:04d}] {name}(ID={cid}) score={payload['s']:.2f}")
        if self.ser:
            self.ser.write(line.encode('utf-8'))
            self.ser.flush()
```

## 六、通信全流程

```
开发板 10Hz 推理
    ↓
GestureDebouncer (3 帧确认 + 去重)
    ↓ 只发确认的新手势
USBSender.send_detection()
    ↓ pyserial → /dev/stm32
USB CDC BULK OUT
    ↓
STM32 CDC_Receive_FS (中断)
    ↓ \n 分行
parse_line() 极简 JSON 解析
    ↓
g_gesture_ready = 1
    ↓
main() 主循环轮询
    ↓ switch(class_id)
风扇 PWM / 舵机扫描
```
